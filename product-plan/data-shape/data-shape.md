This document defines the core entities, their descriptions, and relationships.

## Entities

### User

A person who accesses SnapStim. Each user has one role (BCBA, RBT, or Caregiver) that determines their permissions throughout the system.

### Client

An individual receiving ABA services. Clients are managed by BCBAs and have programs assigned to them.

### Program

An ABA skill-acquisition program assigned to a client. Programs have a type, mode (teaching vs generalization), and contain stimuli.

### Stimulus

A target item within a program. Stimuli can include text, images, or other media. Generated by AI and reviewed by the BCBA.

### Session

A single run of a program, capturing trial-by-trial outcomes. Sessions are run by RBTs or caregivers.

### Trial

An individual trial within a session, recording the outcome for one stimulus.

### Comment

A note attached to a program for communication. Comments have visibility rules based on the author's role.

### Report

A generated report summarizing session data over time. Used for clinical decision-making.

## Relationships

- User (BCBA) manages Client
- User (BCBA) creates Program
- Client has many Program
- Program has many Stimulus
- Program has many Session
- Session belongs to Program
- Session belongs to Client
- Session run by User (RBT or Caregiver)
- Session has many Trial
- Trial records outcome for Stimulus
- Program has many Comment
- Comment authored by User
- Report generated for Client or Program
